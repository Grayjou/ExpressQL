Contributing to Changelog
--------------------------

When contributing changes, please update the CHANGELOG.md file following these guidelines:

1. Add new changes under an "Unreleased" section at the top
2. Categorize changes as:
   - **Added**: New features
   - **Changed**: Changes in existing functionality
   - **Deprecated**: Features that will be removed
   - **Removed**: Removed features
   - **Fixed**: Bug fixes
   - **Security**: Security improvements

3. Keep entries concise but descriptive
4. Link to relevant issues or pull requests

Example entry::

   ## [Unreleased]
   
   ### Added
   - New parsing function for complex expressions (#123)
   
   ### Fixed
   - Fixed NULL handling in conditions (#124)
